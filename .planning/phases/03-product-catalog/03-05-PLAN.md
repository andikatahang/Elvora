STEP A — Add state and methods to the existing Alpine x-data object:

Add a computed getter `averageRating` that calculates the mean of all fetched ratings:
Return `(this.reviews.reduce((s, r) => s + r.rating, 0) / this.reviews.length).toFixed(1)` or `'0.0'` if no reviews. Note: this is the average of loaded reviews only; to show an accurate total the loadReviews method fetches all reviews in the first call for the rating calculation, then uses range() for display — see below.

Add these methods to the Alpine x-data object (called from within init() after `this.loading = false`):

`async loadPairings()`: Query `supabase.from('product_pairings').select('paired_product_id, display_order, products!product_pairings_paired_product_id_fkey(id, name, slug, base_price, product_images(url, alt_text, display_order, colour))').eq('product_id', this.product.id).order('display_order').limit(3)`. Map results to flat pairing card objects. Set `this.pairings`.

`async loadRelated()`: Query `supabase.from('collection_products').select('product_id').eq('collection_id', firstCollectionIdForProduct)` — but this is complex. Simpler approach: query `products` with the same `categories.id` as `this.product.categories?.id` (or fallback: same `category_id` from schema): `supabase.from('products').select('id, name, slug, base_price, product_images(url, alt_text, display_order, colour), product_variants(colour, colour_hex)').eq('category_id', this.product.category_id).eq('is_active', true).neq('id', this.product.id).limit(4)`. Set `this.relatedProducts`.

`async loadReviews()`: Set `this.reviewsLoading = true`. Query: `supabase.from('reviews').select('id, reviewer_name, rating, review_text, fit_feedback, created_at', { count: 'exact' }).eq('product_id', this.product.id).order('created_at', { ascending: false }).range(this.reviewsOffset, this.reviewsOffset + this.reviewsPageSize - 1)`. Append data to `this.reviews`; increment offset by pageSize; update `this.reviewsTotal` from `count`; set `this.allReviewsLoaded = this.reviews.length >= this.reviewsTotal`. Set `reviewsLoading = false`.

Note on averageRating accuracy (Pitfall 7): The loadReviews approach loads 5 at a time, so averageRating will reflect only loaded reviews. For an assessment project this is acceptable — add a code comment `// Note: averageRating reflects loaded reviews only. Phase 7 enhancement: separate aggregate query for accuracy.`

Update `init()` to call after `this.loading = false`:
`await Promise.all([this.loadPairings(), this.loadRelated(), this.loadReviews()]);`

| Supabase reviews data → DOM | Review text and reviewer names from Supabase rendered as text content |
| Supabase product_pairings → PDP cards | Pairing product names/images from Supabase rendered as text/src |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-19 | Tampering | XSS via review_text, reviewer_name | mitigate | All review data rendered via Alpine `x-text` — text node, not innerHTML. XSS-safe even if review text contains script tags |
| T-03-20 | Tampering | product_pairings joined product data | accept | product_pairings is admin-seeded data (RLS admin_write policy); public can only SELECT — no user-submitted pairing data |
| T-03-21 | Information Disclosure | RLS on reviews table | accept | reviews table has RLS from 001_schema.sql (public read, admin write per Phase 1 schema) — no private data exposed |
| T-03-22 | Tampering | Pairing card href injection via product slug | accept | Slug from Supabase products table (admin-controlled) rendered via Alpine :href binding — Alpine escapes attribute values; no user-supplied URL |
</threat_model>
