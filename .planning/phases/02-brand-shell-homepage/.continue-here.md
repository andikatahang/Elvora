# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: Alpine race condition — `type="module"` scripts (components.js) execute AFTER `defer` scripts (Alpine CDN). `alpine:init` fires before components.js loads. Always register Alpine.store() both inside the `alpine:init` listener AND immediately at module execution time. Never rely on `alpine:init` alone.
- [ ] CONSTRAINT: Tailwind `md:` breakpoint (768px) is too narrow for the 5-link nav — the desktop links + action icons overflow at that width. All nav show/hide breakpoints use `lg:` (1024px), not `md:`.
- [ ] CONSTRAINT: `js/__env.js` is gitignored and must NEVER be committed. It is created at Netlify build time from env vars. Locally it contains empty strings, which causes Supabase to return a no-op stub (graceful degradation, not a crash).

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Store registration only in alpine:init | Registering Alpine stores only inside the `alpine:init` event misses the window where defer-loaded Alpine fires the event before the module executes | blocking | Always call `Alpine.store()` directly at module load AND inside `alpine:init` |
| Committing js/__env.js | File holds Supabase credentials and is in .gitignore; staging it exposes credentials | blocking | Verify .gitignore includes `js/__env.js` before every commit touching js/ |
| Using md: breakpoints for nav | 5 links + cart/search/account icons at md: (768px) overflows nav container | advisory | Always use lg: (1024px) for nav desktop/mobile toggle |
